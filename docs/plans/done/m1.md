## Goal

Get the MCP server working end-to-end as a drop-in replacement for augi-engine-v1: Claude can read the vault graph and write documents back to it. Registered and working in Claude Desktop and Claude Code.

---

## What Shipped

### Write tools

- **`write_document(title, content, subfolder)`** — creates `.md` in `{vault_path}/OpenAugi/{subfolder}/`. Agent picks subfolder (Docs, Notes, Research, etc.). Path traversal blocked at `VaultWriter._resolve_folder()`. Collision-safe. Auto-generates frontmatter (type, created timestamp).
- **`reload_index()`** — force-refreshes FAISS by resetting `_faiss_index = None` and `_db_mtime = 0`. Normally auto-reloads on DB mtime change; this is the manual escape hatch.

### Resource

- **`vault://note/{title}`** — dynamic resource. Returns all entries for a note + inbound/outbound link counts. Available as `@openaugi:vault://note/Title` in Claude Code autocomplete.

### Vault path wiring

- `_get_vault_path()` in server — reads `OPENAUGI_VAULT_PATH` env var, falls back to `config.toml [vault] default_path`.
- `openaugi init` already writes `default_path` to config; env var is the override for non-default setups.

### Refactors

- Extracted `_get_embedding_model()` helper — removes `global _embedding_model` duplication from `search()` and `get_context()`.
- Fixed silent `except Exception: pass` in `get_context` semantic search → now logs warning.

### Docs

- `docs/MCP_SERVER.md` — Claude Desktop JSON config, `claude mcp add` command, env vars reference, tools table, write scope explanation, troubleshooting.

## Design Decisions

| Decision | Choice | Why |
|---|---|---|
| Write scope | `OpenAugi/` root only | Safety — agent output stays separate from user's own notes |
| Subfolder | Agent-chosen | Agent knows context (research vs notes vs docs); we just enforce the root |
| Writes to disk only | No DB insert on write | Next `openaugi ingest` picks up new files naturally. No special-case logic. |
| Vault path source | Env var > config.toml | Env var is the override for multi-vault or CI setups; config is the convenience default |
